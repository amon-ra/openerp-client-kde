Chapter 2. Installation

2.1. Common configuration for all platforms

Note that KTiny is intended to work with TinyERP 4.2 server or newer.

Although KTiny tries to work without any server modification, at least, one is necessary at the moment. We'll try to provide a patch in the future but, please follow these simple instructions:

  1. Edit the bin/addons/base/ir/ir_attachment.py file of the server and remove readonly=True from res_model and res_id. Then reestart the server and now you can fully work with attachments within KTiny. Note that this doesn't affect other attachments you may have in the database, and all of them should be available in KTiny. This change is needed for KTiny to create new attachments.

  2. Also, once you've created a new database (or in the original XML file) you should modify the ir.attachment.view of type form. You'll need res_model and res_id to have invisible="1" nolabel="1" as attributes. Also add fname_widget="datas_fname" as attribute to the datas field. The resulting XML should look like this: 

    <form string="Attachments">
	<field colspan="4" name="name" select="1"/>
	<field colspan="4" name="datas" fname_widget="datas_fname" />
	<field name="datas_fname"/>
	<newline/>
	<field name="res_model" select="1" invisible="1" nolabel="1"/>
	<field name="res_id" invisible="1" nolabel="1"/>
	<separator colspan="4" string="Description"/>
	<field colspan="4" name="description" nolabel="1"/>
<form>
